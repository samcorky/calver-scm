# Security Policy

## Supported versions

Security fixes are provided on a best-effort basis for:

| Version                  | Supported |
|--------------------------|-----------|
| Latest published release | Yes       |
| `main` branch            | Yes       |
| Older releases           | No        |

## Reporting a vulnerability

Please do not report security issues in public issues or pull requests.

Report privately via GitHub Security Advisories:

https://github.com/samcorky/calver-scm/security/advisories/new

If you cannot use that channel, open a minimal public issue requesting a private
contact method and do not include sensitive details.

## What to include

To help maintainers triage quickly, include:

- A clear description of the issue and likely impact
- Affected versions and environment details
- Reproduction steps or a minimal proof of concept
- Any suggested remediation or mitigation

## Response expectations

- We aim to acknowledge reports promptly.
- We investigate and assess severity as quickly as possible.
- We may ask for extra details to reproduce and validate the issue.
- We coordinate disclosure timing with reporters when possible.

## Disclosure

Please allow time for investigation and a fix before public disclosure.

When a fix is available, maintainers may publish an advisory with impacted
versions, fixed versions, and mitigation guidance.
